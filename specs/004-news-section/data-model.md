# Data Model: News Section

**Feature**: News Section | **Date**: March 29, 2026  
**Phase**: 1 - Data Model Design

## Entity Overview

This feature introduces a single primary entity: **News**. The entity supports bilingual content (English/Arabic), dual storage for images (Blob/Local), soft deletes, and visibility controls.

## Entity: News

### Purpose
Stores news items displayed on the home page and dedicated news listing page. Supports bilingual titles, dual-storage images, publication scheduling, and soft deletion.

### Fields

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| **id** | String | PK, cuid() | Unique identifier |
| **title_en** | String | Nullable | English news title |
| **title_ar** | String | Nullable | Arabic news title |
| **image_url** | String | Required | Image URL (Blob) or API endpoint (Local) |
| **storage_type** | String | Default: "blob" | Storage method: "blob" or "local" |
| **file_data** | Bytes | Nullable | Binary image data (for local storage) |
| **file_name** | String | Nullable | Original filename (for local storage) |
| **file_size** | Int | Nullable | File size in bytes (for local storage) |
| **mime_type** | String | Nullable | MIME type (e.g., image/jpeg) |
| **published_date** | DateTime | Required | Publication date/time |
| **is_visible** | Boolean | Default: true | Visibility control |
| **is_deleted** | Boolean | Default: false | Soft delete flag |
| **created_at** | DateTime | Auto (now()) | Creation timestamp |
| **updated_at** | DateTime | Auto (updatedAt) | Last update timestamp |

### Field Validation Rules

**title_en / title_ar**:
- At least one must be non-empty
- Max length: 500 characters each
- No HTML tags allowed (sanitize on input)

**image_url**:
- Required field
- For blob storage: Must be valid HTTPS URL (Vercel Blob domain)
- For local storage: Format `/api/news/media/{id}`

**storage_type**:
- Enum: "blob" | "local"
- Default: "blob"
- Cannot be changed after creation (immutable)

**file_data / file_name / file_size / mime_type**:
- Required when `storage_type === "local"`
- Must be null when `storage_type === "blob"`
- Max file_size: 10485760 bytes (10MB)
- Allowed mime_types: image/jpeg, image/png, image/gif

**published_date**:
- Cannot be more than 1 year in the future
- Can be backdated for historical news

**is_visible**:
- Controls whether news appears on public pages
- Admin can toggle on/off
- Default: true

**is_deleted**:
- Soft delete flag (never hard delete)
- Hidden from all queries except admin "trash" view
- Default: false

### Indexes

**Primary Index**:
- `id` (auto-generated by Prisma)

**Composite Index** (Query Optimization):
```prisma
@@index([is_deleted, is_visible, published_date(sort: Desc)])
```

**Rationale**:
- Most queries filter by `is_deleted: false` and `is_visible: true`
- Results always ordered by `published_date DESC`
- Composite index covers all three fields in query order
- Expected performance: <50ms for 1000+ news items

**Query Pattern Example**:
```sql
SELECT * FROM news
WHERE is_deleted = false
  AND is_visible = true
ORDER BY published_date DESC
LIMIT 12 OFFSET 0;
-- Uses: idx_news_is_deleted_is_visible_published_date
```

## Prisma Schema Definition

```prisma
model News {
  id             String    @id @default(cuid())
  title_en       String?   @db.Text
  title_ar       String?   @db.Text
  image_url      String    @db.Text
  storage_type   String    @default("blob") @db.VarChar(10)
  file_data      Bytes?    @db.ByteA
  file_name      String?   @db.VarChar(255)
  file_size      Int?
  mime_type      String?   @db.VarChar(50)
  published_date DateTime  @db.Timestamptz(6)
  is_visible     Boolean   @default(true)
  is_deleted     Boolean   @default(false)
  created_at     DateTime  @default(now()) @db.Timestamptz(6)
  updated_at     DateTime  @updatedAt @db.Timestamptz(6)

  @@index([is_deleted, is_visible, published_date(sort: Desc)])
  @@map("news")
}
```

## State Transitions

### News Lifecycle

```
[Created] → is_visible: true, is_deleted: false
   ↓
[Published] → Visible on home page (if published_date ≤ NOW)
   ↓
[Hidden] → is_visible: false (still exists, not shown)
   ↓
[Soft Deleted] → is_deleted: true (archived, recoverable)
   ↓
[Restored] → is_deleted: false (back to Hidden or Published)
```

### Admin Actions

| Action | State Change | Effect |
|--------|-------------|--------|
| Create | `is_visible: true, is_deleted: false` | Immediately visible if `published_date ≤ now()` |
| Hide | `is_visible: false` | Removed from public pages, stays in admin |
| Show | `is_visible: true` | Re-appears on public pages |
| Delete | `is_deleted: true` | Moves to admin "trash" view |
| Restore | `is_deleted: false` | Returns to normal admin view |
| Edit | `updated_at` updated | No state change |

---

## Relationships

**No relationships** to other entities. News is a standalone entity.

**Rationale**:
- Simple, self-contained model
- No user authorship tracking needed (admin action logs via Clerk)
- No categories/tags (out of scope for MVP)
- No comments/reactions (future enhancement)

## Data Integrity Constraints

### Application-Level Validation

```typescript
// lib/types.ts
export const NewsSchema = z.object({
  title_en: z.string().max(500).optional().nullable(),
  title_ar: z.string().max(500).optional().nullable(),
  image_url: z.string().url().or(z.string().startsWith('/api/news/media/')),
  storage_type: z.enum(['blob', 'local']),
  file_data: z.instanceof(Buffer).optional().nullable(),
  file_name: z.string().max(255).optional().nullable(),
  file_size: z.number().max(10485760).optional().nullable(),
  mime_type: z.enum(['image/jpeg', 'image/png', 'image/gif']).optional().nullable(),
  published_date: z.date(),
  is_visible: z.boolean().default(true),
  is_deleted: z.boolean().default(false),
}).refine(data => data.title_en || data.title_ar, {
  message: "At least one title (English or Arabic) is required",
});

// API route validation
export async function POST(request: Request) {
  const body = await request.json();
  const validated = NewsSchema.parse(body);
  // ... create news
}
```

- Primary key uniqueness (id)
- NOT NULL constraints on required fields
- Index integrity maintained by PostgreSQL

## Performance Considerations

**Expected Load**:
- ~100-500 total news items
- ~5-10 new items per month
- ~1000 reads/day on home page
- ~100 reads/day on dedicated news page

**Query Performance**:
- Indexed queries: <50ms
- Image serving (local): <100ms (with caching)
- Pagination: <200ms per page

**Storage Estimates**:
- Average image size: 200KB
- 500 news items × 200KB = 100MB total
- PostgreSQL row size: ~500 bytes (metadata)
- Total DB size: ~100MB (well within Neon limits)

## Conclusion

The News data model is simple, self-contained, and follows established patterns from the SliderContent model. It supports all required features (bilingual, dual-storage, pagination, visibility) while maintaining performance and data integrity.
